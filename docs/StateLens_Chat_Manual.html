1. どういうアプリか

State Lens : Chat は、Mac上の iMessage を監視してサーバーへ転送し、Viewer で分析つき表示するアプリです。

できること

  • iMessageの新着を自動取得
  • 感情・意図・話題を自動分析
  • 参加者ごとのタスクを表示
  • thread単位メトリクスの表示

主な構成

  • Relay: imessage_relay.py
  • Viewer API: viewer_server.py
  • UI: viewer/
  • 起動: StateLens_Chat 起動.app

2. 画面の見方

  • 上部: threadごとのメトリクス（送信密度、話題密度など）
  • 中央: メッセージカード（送受信、本文、時刻）
  • 右パネル: 感情・意図・表現・自分のタスク・参加者ごとのタスク
  • analysis_source: openai:gpt-5-mini などならAI分析、fallback は代替分析

3. 使い方（起動〜停止）

起動

  1. StateLens_Chat 起動.app を実行
  2. ブラウザで Viewer を開く（通常 http://localhost:8080/）
  3. iMessage を1件送受信して反映確認

停止

  1. StateLens_Chat 停止.app を実行

全件再分析

curl -X POST http://192.168.4.33:8080/api/reanalyze \
  -H "Content-Type: application/json" \
  -d '{"scope":"all"}'

4. 日常運用（毎日やること）

  1. 起動後に analysis_source が openai:... か確認
  2. 反映遅延がある場合は 10〜30秒待つ（解析キュー処理中）
  3. 必要時のみ全件再分析を実行

状態確認コマンド

curl -s http://192.168.4.33:8080/api/messages | python3 -c 'import sys,json;p=json.load(sys.stdin);m=p.get("meta",{});print("provider=",m.get("analysis_provider"),"openai=",m.get("openai_enabled"),"pending=",m.get("pending_count"))'

5. トラブル時の対処

fallbackのまま変わらない

tail -n 120 ~/Library/Logs/imessage_relay_launcher.log
  • 401: APIキー不正/失効
  • timed out: タイムアウト（OPENAI_TIMEOUT を増やす）
  • 400: モデルパラメータ不一致

新着が取れない

cd "/Users/nishiokamahiro/Desktop/Antigravity/iMessage soujushin"
./scripts/diagnose_relay_access.sh
ポイント: authorization denied が出る場合は、実行中の Python 実体にフルディスクアクセスが必要です。

6. よくある質問

Q. 意図（intent）は何種類？

request / report / ask / confirm / complain / share / plan / emotional_expression

Q. participantタスクの判定は？

本文の明示名・あだ名・送受信方向・条件付きルールを使って推定します。

Q. 一人称の置換は？

私/わたし/僕/俺 や 送信者/話者 を、該当メッセージの送信者人物名へ変換します。
